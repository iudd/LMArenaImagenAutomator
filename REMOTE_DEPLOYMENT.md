# 远程服务器部署指南

本文档提供了在远程服务器上部署 LMArenaImagenAutomator 的详细指南。

## 🚀 快速开始

### 1. 控制台登录模式

对于远程服务器部署，建议使用控制台登录模式：

```bash
npm run start -- -login-console
```

### 2. 检查运行状态

使用状态检查脚本监控部署状态：

```bash
npm run check-status
```

## 🔧 配置优化

### 远程服务器配置建议

在 `config.yaml` 中进行以下配置：

```yaml
chrome:
  headless: true  # 必须启用无头模式
  gpu: false      # 禁用GPU加速
  # 其他配置...
```

## 📊 状态监控

### 自动状态报告

程序在控制台登录模式下会：

- 每30秒输出状态报告
- 自动保存登录状态到 `data/login-status.json`
- 监控内存使用情况
- 提供异常处理和恢复机制

### 状态文件位置

- 登录状态：`data/login-status.json`
- Chrome用户数据：`data/chromeUserData/`
- 应用日志：根据日志配置

## 🛠️ 故障排除

### 常见问题

1. **登录失败**
   - 检查网络连接
   - 确认代理配置正确
   - 重新运行登录命令

2. **程序意外退出**
   - 检查系统资源
   - 查看错误日志
   - 确认进程权限

3. **状态检查失败**
   - 确认程序正在运行
   - 检查文件权限
   - 验证目录结构

### 日志监控

```bash
# 实时查看日志（如果配置了日志文件）
tail -f logs/app.log

# 检查进程状态
ps aux | grep "node server.js"
```

## 📋 状态检查脚本功能

`npm run check-status` 提供以下检查：

1. **登录状态检查**
   - 验证登录状态文件存在
   - 检查登录时间有效性
   - 显示当前页面信息

2. **进程状态检查**
   - 检测运行中的node进程
   - 显示PID和资源使用情况

3. **日志检查**
   - 检查日志文件存在性
   - 显示最近的日志条目
   - 监控文件大小

4. **数据目录检查**
   - 验证数据目录结构
   - 检查Chrome用户数据
   - 显示目录大小

## 🔒 安全建议

1. **API密钥管理**
   - 使用强密码生成API密钥
   - 定期轮换密钥
   - 不要在代码中硬编码密钥

2. **网络安全**
   - 配置防火墙规则
   - 使用HTTPS连接
   - 限制API访问IP

3. **进程安全**
   - 使用专用用户运行
   - 限制文件权限
   - 监控异常访问

## 📈 性能优化

1. **资源配置**
   - 确保足够的内存（建议2GB+）
   - 配置适当的CPU限制
   - 监控磁盘空间

2. **浏览器优化**
   - 禁用不必要的扩展
   - 配置合适的超时时间
   - 定期清理缓存

## 🔄 维护建议

1. **定期维护**
   - 每周检查状态文件
   - 清理临时文件
   - 更新依赖包

2. **备份策略**
   - 备份配置文件
   - 备份Chrome用户数据
   - 记录重要配置更改
